# ROOT FIX for the cross-build board divergence (item 391): np.interp on the live
# value path diverges >=1e-8 across DIFFERENT numpy WHEELS (a different compile of
# numpy's compiled_base.c interp routine), amplifying past the ~1e-12 board flip
# threshold and reordering the board RANK-UNSAFELY on a minority of containers.
# np.interp does NOT call BLAS, so the amplifier lives in the numpy WHEEL itself.
# The numpy 2.4.4 wheel also BUNDLES its OpenBLAS (numpy.libs/libscipy_openblas64_
# -32a4b2a6.so), so hash-pinning the numpy wheel pins BOTH the compiled interp code
# AND the BLAS in one artifact — the smallest complete pin.
#
# numpy below is THE determinism-critical pin. The rest of the compiled scientific
# stack is pinned for a fully reproducible env (they cost nothing extra to pin).
#
# INSTALL (hash-verified, wheel-only — an sdist build would recompile numpy and
# reintroduce the divergence, so --only-binary forbids it):
#   pip install --require-hashes --only-binary=:all: -r requirements-lock.txt
#
# Platform: CPython 3.12 / manylinux x86_64 (the standard build container). The
# hashes are for the cp312 manylinux_2_28 x86_64 wheels; a build on any other
# platform HALTS (no matching hash) rather than silently using a divergent wheel.
# Regenerate ONLY when the owner deliberately moves the build env (re-run the
# board reproduction gate against 06d8af60 in the same change).
# ============================================================================
